# CashFlinch Engineering Rules

## Product Identity

CashFlinch is a premium behavioral finance diagnosis app.

The app should feel:
- cinematic
- emotionally intelligent
- premium fintech
- minimalist
- psychologically sharp
- self-aware
- screenshot-worthy

The app should NOT feel:
- meme quiz
- admin dashboard
- overanimated AI SaaS template
- cluttered Tailwind UI
- productivity app

## Architecture Rules

Do not:
- rewrite architecture
- duplicate logic
- create giant monolithic files
- introduce unnecessary abstractions
- move scoring systems unnecessarily
- touch unrelated systems during refactors

Prefer:
- incremental refactors
- reusable components
- modular structure
- minimal complexity
- production-ready code

## Design Rules

Prioritize:
1. whitespace
2. typography hierarchy
3. visual hierarchy
4. emotional clarity
5. responsiveness
6. cinematic composition

Avoid:
- card spam
- excessive gradients
- dashboard clutter
- overuse of serif fonts
- overly dense layouts
- flashy gimmicks
- unnecessary animations

## UX Philosophy

The app should feel like:
“an emotionally intelligent behavioral finance report”

NOT:
“a funny internet quiz”

The roast section is the emotional centerpiece.

Metrics should support the diagnosis, not compete equally.

## Engineering Philosophy

- maintainable Tailwind only
- preserve architecture
- improve incrementally
- avoid unnecessary dependencies
- optimize for MVP quality
- prioritize product polish over complexity
